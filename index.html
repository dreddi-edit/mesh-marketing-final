Quick FAQ
How is Mesh different from a normal coding chat?+

Mesh indexes your codebase into a capsule and retrieves exact spans on demand. That means lower token cost, deterministic references, and fewer invented paths compared to plain full-file chat.

Do I need all four surfaces to start?+

No. Start with the CLI, then add IDE, MCP, or API only when your workflow needs them. All surfaces share the same capsule engine, so context carries across.

Is the IDE available on Windows?+

Current downloadable builds target macOS. Windows support is planned. The stable download URL always serves the latest macOS DMG for now.

Can teams run Mesh in CI?+

Yes. Teams usually combine CLI and API for CI pipelines, while MCP and IDE support day-to-day developer workflows. You can adopt incrementally without rewriting your stack.

Do we have to re-index on every change?+

No. You usually re-index when there are significant structural code changes. For fast iterative edits, the existing capsule is often enough and recover calls stay accurate.

How do updates to the IDE download work?+

The public URL stays stable. New builds overwrite the same Blob path, and /download/ide-macos always redirects to the latest Mesh.dmg.

Can we keep data local?+

Yes. A common setup is local indexing and local recovery with your own provider keys. Teams can decide how much to run local vs cloud per surface.

Where should new users start?+

Start in CLI, run mesh, and build confidence with index/recover flows. Then add MCP for coding assistants and IDE/API only when your team needs those surfaces.
